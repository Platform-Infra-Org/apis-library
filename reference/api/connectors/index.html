    Connectors API¶

    The four infrastructure connectors. Each is a three-layer stack — a high-level service users instantiate, a low-level client, and Pydantic models for requests/responses (see Architecture). Import the services from the top-level package:

    from tashtiot_apis_library import AWX, ArgoCD, Git, Vault

    The clients and models are available from each connector's subpackage, e.g. from tashtiot_apis_library.connectors.awx import AWXClient, AWXJob.

    See Drive AWX, ArgoCD & Vault and Read & write Bitbucket files for task-oriented usage.

    Standardized responses¶

    The base response model returned by service operations (connector-specific subclasses are in the per-connector models below).

    OperationResponse ¶

    Bases: BaseModel

    Base response schema for operations.

    Attributes:

    Name Type Description
    status str

    Operation status - typically "successful", a response from external service, or "failed"
    status_code Optional[int]

    HTTP status code if applicable
    return_code Optional[int]

    Process return code if applicable
    stdout str

    Standard output from the operation

    status class-attribute instance-attribute ¶

    status: str = Field(
    ...,
    description='Operation status: "successful", external service response, or "failed"',
)

    status_code class-attribute instance-attribute ¶

    status_code: Optional[int] = Field(
    default=None,
    description="HTTP status code from external service",
)

    return_code class-attribute instance-attribute ¶

    return_code: Optional[int] = Field(
    default=None, description="Process return code"
)

    stdout class-attribute instance-attribute ¶

    stdout: str = Field(
    default="",
    description="Standard output from the operation",
)
